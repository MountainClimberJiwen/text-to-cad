# Agent Runtime 文档路由（浅 -> 深）

## Level 0：全局理解

先读取 `../WORKFLOW.MD`，快速获得：

- 8 个模块的目的
- 关键参数与约束
- 项目周期与验证边界

适用场景：用户问题不明确、需要先做任务分发或确定模块归属。

## Level 1：模块定位

根据问题类型进入对应模块文件：

- 需求/能力边界：`../modules/01_设备需求分析.md`
- 工序链路：`../modules/02_产品工艺流程.md`
- 物料与上料：`../modules/03_物料管理.md`
- 节拍与空间：`../modules/04_设备布局.md`
- 机构动作与检测：`../modules/05_详细机构设计.md`
- 选型与品牌标准：`../modules/06_核心元器件选型.md`
- 合规验证与文件：`../modules/07_项目管理与验证体系.md`
- 排期与里程碑：`../modules/08_交货期规划.md`

## Level 2：操作细化

在模块文件内优先查阅“可扩展操作清单”：

- 若已有步骤：按步骤执行并记录结果
- 若缺少步骤：在对应模块追加“操作步骤”小节，按输入/输出补齐

## Level 3：跨模块联动

当问题跨模块时，按顺序联动：

1. `01_设备需求分析` 定义目标与边界
2. `02_产品工艺流程` 确认工序链
3. `05_详细机构设计` 落地动作与检测
4. `07_项目管理与验证体系` 验证合规闭环
5. `08_交货期规划` 评估可交付时间

## 维护规则

- 每次新增操作细节，必须补充输入/输出与验收标准
- 模块内新增内容优先最小闭环，避免一次性写大而全
- 若模块边界变化，先改 `WORKFLOW.MD` 总览再改模块细节
